# Koray Holistic Writing Optimizer

## Always load

- `references/koray-writing-rules.md`

## Workflow

1. Identify the target content and its primary query/task per section (H2/H3).
2. Rewrite each section to lead with an immediate factual answer (1–3 sentences), then expand with evidence, constraints, and examples.
3. Enforce all 42 rules — run the full checklist; fix every failure before marking complete.
4. Produce the outputs.

## Rules checklist (all 42 — full details in `references/koray-writing-rules.md`)

**Core rules 1–14 (apply to every sentence):**

1. Proper word sequence — most important information first
2. Be certain and factual — no hedging
3. Cut all fluff and metadiscourse
4. Use numeric values and qualifiers when relevant
5. Qualify instances — examples after every plural noun
6. Correct verb context — use precise verbs
7. Examples after plural nouns
8. Same POS in all lists
9. Immediate factual answer under every heading — no delay
10. Bold the answer, not the search term
11. One macro context per page — never drift
12. "If" statements in second part of sentence
13. Anchor text matches destination page title exactly
14. Include relevant entities and semantically related terms

**Advanced patterns 15–42 (apply per section):**

15. Paragraph function: claim → evidence → implication
16. Subordinate text self-contained in first sentence
17. No negative phrases — rephrase positively
18. Positive and empathetic tone
19. EAV triple integration (entity/attribute/value naturally embedded)
20. Verbs-of-life flow (discover → learn → apply → achieve)
21. Source context on every claim ("According to…", "As of [date]…")
22. Information gain — add unique EAV competitors don't have
23. Heading vector alignment — heading matches user query intent exactly
24. Safe answer principle — evidence-based, no overpromises
25. Numeric density 1–2 per 100 words
26. List optimization — max 7 items, introduce with purpose sentence
27. Internal linking — contextual, in the most relevant section
28. Title and meta description alignment with H1 intent
29. URL structure: `/central-entity/primary-attribute/`
30. Schema readiness — content supports at least one schema type
31. Readability — average 18–22 words per sentence
32. No contextless words or orphaned pronouns
33. Historical data signals where relevant
34. User behavior alignment — answer what the user does next
35. Momentum building — each section leads to the next logically
36. Depth layering — surface answer → supporting detail → edge cases
37. Vastness support — reference related entities and bridge topics
38. Updateness — "as of [current date]" on time-sensitive facts
39. Evidence-based — no invented numbers or claims
40. Algorithmic authorship signals — consistent expert voice
41. Final polish — read-aloud test; fix awkward phrasing
42. HCU compliance — page fully completes the user's task

## Output

- **Optimized version** (full rewrite in Markdown)
- **Compliance table**: `Rule # | Rule name | Pass/Fail | Fix applied`
- **High-impact diffs** (top 5–10 changes, not a full diff)
- **Title / meta description / URL suggestions** (only if heading structure changed)
